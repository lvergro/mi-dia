## Context
El proyecto tiene una app mobile (Expo) y una web futura (Next.js) que comparten tipos, lógica de dominio y configuración. Un monorepo permite compartir código sin publicar paquetes.

## Decision
Usar pnpm workspaces para gestión de dependencias y Turborepo como orquestador de builds y tareas (test, lint, build) con caché remota opcional.

## Options considered
| Option | Pros | Cons |
|--------|------|------|
| pnpm + Turborepo | caché de builds, pipelines declarativos, bien integrado con Vercel | configuración inicial |
| pnpm workspaces solo | simple, sin overhead | sin caché ni pipelines |
| Nx | muy completo, generadores de código | más complejo, mayor curva de aprendizaje |
| yarn workspaces + Lerna | maduro | pnpm es más eficiente en disco y velocidad |

## Consequences
- Positive: caché de builds acelera CI, un solo repositorio para mobile + web + paquetes compartidos.
- Negative: Turborepo añade una capa de configuración (`turbo.json`).
- Neutral: integración nativa con Vercel (detecta Turborepo automáticamente en deploy).
