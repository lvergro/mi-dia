# Research — Mi Día

## 2026-05-04 — Stack inicial seleccionado

**Problem**: Elegir stack para MVP mobile + backend simple.

| Option | Pros | Cons |
|--------|------|------|
| Expo + Supabase | Rápido, BaaS completo, RLS nativo, Auth incluido | Vendor lock-in parcial |
| React Native CLI + Firebase | Más control | Más setup, sin RLS nativo |
| Flutter + Supabase | Performance nativa | Cambio de lenguaje (Dart) |

**Recommendation**: Expo + Supabase. Menor fricción para MVP, RLS y Auth sin configuración adicional.
